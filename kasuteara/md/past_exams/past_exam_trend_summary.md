# 過去期末テスト出題傾向メモ

対象資料:

- 金子&山井2024期末.pdf
- 山井&金子_2023期末メモ.pdf

注: どちらも記憶頼りの再現メモであり、実際の問題文・条件と完全一致するとは限らない。攻略サイトでは「出題傾向」として扱う。

## 全体傾向

山井先生側は、形式言語・正規表現・オートマトン・構文解析が中心。単なる用語説明ではなく、正規表現を書く、NFA/DFAを変換する、構文木を書く、First/Followを求める、LR(1)のclosureや解析を行う、という手計算問題が出る。

金子先生側は、コード生成・アセンブリ・スタック・最適化・レジスタ数見積もりが中心。C風の短い関数からアセンブリ穴埋め、実行途中のスタック/eax状態、覗き穴最適化後のコード、RSL/RLなどのコード生成アルゴリズム、GC/ICCアルゴリズムの挙動が出る。

## 山井先生側の頻出テーマ

Sランク:

- 正規表現: 0/1上の条件を満たす言語を正規表現で書く。
- ε-NFA/NFA/DFA: ε動作つきNFAからεなしNFA、NFAからDFAへの変換。
- 曖昧な文法と構文木: if-then-else、`i+i*i` などに対して複数の構文木を書く。
- LL(1): First, Follow, Director集合を求める。
- LR(1): closure、状態、解析表または授業課題相当の問題。

Aランク:

- 最右導出。
- EBNFまたは構文図式の相互変換。
- 逆ポーランド記法。
- RL文法または構文解析方式に関する問題。

## 金子先生側の頻出テーマ

Sランク:

- アセンブリコード穴埋め: C関数に対応する命令列を選択肢から埋める。
- スタックとレジスタ状態: `esp`, `ebp`, 引数、戻りアドレス、`eax` の値を追跡する。
- 覗き穴最適化: `shl`, `dec`, 不要なスタックフレーム除去などの局所最適化。
- コード生成アルゴリズム: RSL/RLなどの型、必要レジスタ数、命令型を表から判断する。

Aランク:

- GCアルゴリズム、ICCアルゴリズムの挙動。
- 汎用レジスタ数が制約された式の評価。
- `if` 条件分岐を含む短い関数のコード生成。

## 2024メモからの具体例

山井先生:

- `00` または `11` を含む正規表現。
- `00`, `11` を含まない正規表現。
- 最右導出。
- `S -> if C then S | if C then S else S` に対する `if C then if C then S else S` の2通りの構文木。
- `First(T)`, `Follow(T')`。
- `E -> E + . T` のclosure。
- ε動作なしNFAをDFAへ変換。
- 逆ポーランド記法で `(A-B^(C^D))-(E*F)` を示す。

金子先生:

- `int foo(int a, int b){ if(a >= 0 && a < b) b = a; return b; }` に対するアセンブリ穴埋め。
- L3まで実行したときのスタックと `eax` の値。
- 最適化後のアセンブリ穴埋め。
- `{(a-1)*(b-2)+(c-3)*(e-4)*(d-5)}*{(e)-(f-6)}` の推測レジスタ数と命令型。

## 2023メモからの具体例

山井先生:

- 4の倍数となる2進数の正規表現。
- 0と1をどちらも含む数の正規表現。
- ε動作つきNFAをε動作なしNFAへ変換。
- 構文図式からEBNFを書く。
- `i+i*i` を表す2つの構文木。
- `E -> T E'`, `E' -> + T E' | ε`, `T -> F T'`, `T' -> * F T' | ε`, `F -> (E) | i` について `First(E')`, `Follow(T)` を求める。
- LR(1)文法の回の課題相当。

金子先生:

- `f(a,b){ int c=a*2-1; return c+b; }` から得られるアセンブリコード穴埋め。
- ラベルL1直前のスタックと `eax` レジスタ状態。
- 覗き穴的最適化後のコード穴埋め。
- RSL型、RL型などのコード生成アルゴリズムと必要レジスタ数。
- GCアルゴリズムとICCアルゴリズムの挙動。

## 攻略サイトへの反映方針

- 「最重要演習」には、正規表現、NFA/DFA変換、曖昧な文法、First/Follow、LR(1)、アセンブリ穴埋め、スタック追跡、レジスタ数見積もりを必ず入れる。
- 各テーマは、授業資料の対応ページ画像と過去問メモの根拠を並べる。
- 過去問メモ由来の内容には「記憶再現メモに基づく」と明記する。
- 金子先生側は、図だけでなく「手で追う表」を作る。特にスタック、`eax`、ラベル到達時点の状態を段階的に追えるようにする。
- 山井先生側は、解法テンプレートを重視する。正規表現の作り方、ε-closure、subset construction、First/Follow計算、closure計算を手順化する。
