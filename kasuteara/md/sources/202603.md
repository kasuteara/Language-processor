TMの停止性問題（３）
• 任意のTM 𝑀𝑀と入力𝑤𝑤が与えられた場合，「有限時間
で停止するか停止しないか」を判定するTMは
存在するか（停止性問題）⇒存在しない
• そのようなTM 𝐻𝐻(𝑀𝑀, 𝑤𝑤)が存在すると仮定・・・①
• 𝑀𝑀(𝑤𝑤)が停止する ⟺𝐻𝐻𝑀𝑀, 𝑤𝑤= YES（受理）
• 𝑀𝑀(𝑤𝑤)が停止しない⟺𝐻𝐻𝑀𝑀, 𝑤𝑤= NO（不受理）
• 次のような新しいTM 𝑀𝑀𝑀(𝐴𝐴)を構成
• 𝐻𝐻A, A = YES ⟺𝑀𝑀′(𝐴𝐴)が停止しない（無限ループ）
• 𝐻𝐻(𝑀𝑀, 𝑤𝑤)の受理状態で無限ループになるように修正（容易に可能）
• 𝐻𝐻A, A = NO ⟺𝑀𝑀′(𝐴𝐴)は停止
• 𝑀𝑀′(𝑀𝑀𝑀)の停止性を検討するといずれにせよ矛盾
• 𝑀𝑀′(𝑀𝑀′)が停止⇒𝐻𝐻𝑀𝑀𝑀, 𝑀𝑀𝑀= NO ⇒𝑀𝑀𝑀(𝑀𝑀𝑀)は停止せず⇒矛盾
• 𝑀𝑀′(𝑀𝑀′)が停止せず⇒𝐻𝐻𝑀𝑀′, 𝑀𝑀′ = YES ⇒𝑀𝑀′(𝑀𝑀′)は停止⇒矛盾
• 背理法により①の仮定が間違い⇒ 𝐻𝐻(𝑀𝑀, 𝑤𝑤)は存在しない

計算複雑性理論（５）
• 𝑁𝑁𝑁𝑁困難問題の例
• 巡回セールスマン問題（Traveling salesman problem）
• 入力：都市の集合と各2都市間の移動コスト（たとえば距離）
• 出力：各都市を1回ずつ訪問して出発地に戻る最小コスト経路
※ 出力が「コスト𝐿𝐿以下の巡回経路があるかどうか」（決定問題）
であれば𝑁𝑁𝑁𝑁完全
• ナップサック問題（Knapsack problem）
• 入力：容量𝐶𝐶のナップサック，𝑛𝑛種類の荷物（価値𝑝𝑝𝑖𝑖，容積𝑐𝑐𝑖𝑖）
• 出力：合計容積𝐶𝐶以下で合計価値を最大とする荷物の組合せ
 ※ 出力が「合計価値が𝑊𝑊以上となる組合せがあるかどうか」
（決定問題）であれば𝑁𝑁𝑁𝑁完全

正規表現（５）
• 正規表現と字句解析との関係
• 多くの場合，トークンは正規表現で定義可能
• 例： C言語の識別子は[_A-Za-z][_A-Za-z0-9]*
• 英字（アンダースコアを含む大小文字）から始まる英数字列
• 正規表現で定義可能だが簡単ではないもの
• 例： C言語のコメント「/* ・・・ */」
• 「/¥*.*¥*/」は不可（longest matchのため）
• 「/* コメント1 */ a=1; /* コメント2 */」全体がコメント
• 「 ・・・ 」に相当する，“*/” 以外の文字列
• 「([^*]|¥*+[^*/])*¥**」で定義可能だが直観的でない
• PCREでは「/¥*.*?¥*/ 」で表現可能（*?: lazy matchingでの0回以上）
• 正規表現だけでは困難なもの
• 古いFORTRAN（基本的には空白文字は無意味）
• DO␣10␣I = 1,5    →   Iを1～5に変化させて文番号10まで繰返し
• DO␣10␣I = 1.5    →   「DO10I」という実数型変数に1.5を代入

• 形式的な定義
• 𝑀𝑀= (𝑄𝑄, 𝛴𝛴, 𝛿𝛿, 𝑞𝑞0, 𝐹𝐹) の5つ組で定義

• 𝛿𝛿: 遷移関数（𝑄𝑄× 𝛴𝛴→𝑄𝑄）
•
入力：（現在の状態，次の入力記号）
•
出力：次の状態
• 𝑞𝑞0: 初期状態（𝑞𝑞0 ∈𝑄𝑄）
• 𝐹𝐹: 受理状態（𝐹𝐹⊆𝑄𝑄）
